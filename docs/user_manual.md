# 学业规划助手 — 用户手册

## 快速开始

### 命令行模式

```bash
cd study_agent
python app.py
```

### Web 界面模式

```bash
cd study_agent
pip install streamlit plotly pandas
streamlit run streamlit_app.py
```

### 前置条件

1. Python 3.10+
2. 安装依赖: `pip install -r requirements.txt`
3. Redis（可选）: 不装也能正常使用，系统自动降级为内存存储
4. LLM API Key（可选）: 不配也能用，系统使用规则模板，功能不受影响

## 功能列表

| 功能 | 指令示例 | 说明 |
|------|---------|------|
| 推荐辅修 | `推荐辅修` / `我想选辅修` | 根据兴趣方向推荐辅修专业及课程方案 |
| 查看进度 | `查看进度` / `成绩查询` | 查看学习进度、作业/小测趋势和出勤率 |
| 检测冲突 | `检测冲突` / `时间冲突` | 检查主修与辅修课表的时间重叠 |
| 推荐资源 | `推荐资源` / `找资料` | 根据薄弱课程推荐学习资料（习题/笔记/网课） |
| 生成报告 | `生成报告` / `学业报告` | 生成综合学业报告并导出 .md 文件 |
| 我的课表 | `我的课表` / `查看课表` | 按星期展示完整课程安排 |
| 录入课表 | `录入课表` / `添加课程` | 逐门添加主修课程信息（课程代码+时间+教室） |
| 退课/换课 | `退课` / `换课` | 细粒度调整辅修课程，换课自动预检冲突 |
| 调课 | `调课` | 一键清空辅修课程，重新规划 |
| 帮助 | `帮助` | 查看所有可用功能及指令 |

## 典型使用流程

### 第一步: 录入主修课表

```
> 录入课表
📝 数据录入模式
请按以下格式输入主修课程信息...

> CS101 周一1-3节 教三楼201
✅ 已添加 CS101

> 完成
✅ 数据录入完成
```

### 第二步: 选择辅修

```
> 推荐辅修
为您推荐以下辅修方案...

> 选方案一
✅ 已选择 金融学 辅修
```

### 第三步: 检测冲突

```
> 检测冲突
共检测到 2 条时间冲突...
```

### 第四步: 跟踪进度

```
> 录入成绩

> W5 CS201 作业90 小测85 出勤
✅ 第5周数据已记录
```

### 第五步: 查漏补缺

```
> 推荐资源
分析薄弱课程...为您推荐以下学习资料...
```

### 第六步: 生成报告

```
> 生成报告
📊 学业综合报告已生成
📁 报告已导出: data/exports/report_u001_20260615_143022.md
```

## 配置 LLM 增强（可选）

设置环境变量后，报告生成和学科问答质量显著提升：

```bash
# Windows PowerShell
$env:ANTHROPIC_API_KEY="sk-ant-..."

# Linux / macOS / Git Bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

不设置也能正常使用所有功能，系统自动回退到规则模板。

## 常见问题

**Q: 没有 Redis 能运行吗？**
A: 能。系统检测到 Redis 不可用时自动切换为内存存储，多轮对话状态在进程内保持。

**Q: 没有 API Key 能用吗？**
A: 能。所有功能都有完整的规则/模板回退。LLM 只是增强，不是必需。

**Q: 报告文件保存在哪？**
A: `data/exports/` 目录，Markdown 格式。

**Q: 如何修改已录入的课表？**
A: 输入 `删除课程` 删除错误条目，再 `录入课表` 重新添加。
